# 0001. B2B v1 — мінімальний продаваний скоуп

- **Статус:** accepted
- **Дата:** 2026-06-10
- **Автор:** Roman Oliakh (за специфікацією партнера `Learning Platform.xlsx`)
- **Затвердив:** зафіксовано до запровадження decision log; скоуп живе в CLAUDE.md «Product scope — B2B v1» (PR #6, merged)

## Контекст

Партнер надіслав специфікацію на 50 флоу / 30 модулів (повноцінний корпоративний
LMS). Будувати все одразу — 6–12 місяців без продажів. Продукт — партнерство;
мета — якнайшвидше вийти на перших платних пілотів.

## Розглянуті варіанти

1. Повна спека одразу — довго, дорого, без раннього доходу.
2. B2C-first (лишити магазин курсів) — не відповідає моделі партнера.
3. **B2B-first, мінімальний продаваний зріз** — компанія купує доступ →
   онбордить людей → призначає курси → бачить звіт. ✅ обрано.

## Рішення

v1 = tenancy + ручне провіжування компаній (sales-led) + інвайти співробітників
+ призначення курсів + learner-досвід (reuse) + **звіт по компанії з CSV** +
простий PDF-сертифікат. Ролі: platform_admin / company_admin / learner.
Повні IN/OUT-списки: CLAUDE.md «Product scope — B2B v1» (канонічний перелік).

## Наслідки

- Teams/відділи, Manager/Instructor, Programs/Journeys, воркшопи, завдання,
  self-serve білінг, B2C, AI, SSO, SCORM, BI — свідомо НЕ в v1.
- Білінг v1 — ручний інвойс (без Stripe-підписок).
- Глибші рішення моделі даних чекають P0-відповідей у `docs/discovery-questions.md`.
