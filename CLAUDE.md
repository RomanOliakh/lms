# LMS Project

## Stack

- **Next.js 16** (App Router, Turbopack) — read `node_modules/next/dist/docs/` before writing any code
- **React 19**
- **TypeScript 5**
- **Tailwind CSS v4** — no `tailwind.config.ts`; all theme customization lives in `@theme inline` inside `app/globals.css`
- **shadcn/ui 4** — components in `components/ui/`, registry via `npx shadcn@latest add <name>`
- **Supabase** — `@supabase/supabase-js` + `@supabase/ssr`; server client in `lib/supabase/server.ts`, browser client in `lib/supabase/client.ts`
- **Stripe** — `stripe` (server) + `@stripe/stripe-js` (client)
- **Resend** — transactional email
- **lucide-react** — icons
- **clsx** + **tailwind-merge** — use via `lib/utils.ts` `cn()`

## Commands

```bash
npm run dev        # start dev server (Turbopack, default port 3000)
npm run dev -- -p 3001  # alternate port if 3000 is taken
npm run build      # production build — also type-checks (no separate tsc step)
npm run lint       # ESLint
```

No test runner is configured — `npm test` will fail.

Regenerate Supabase types after schema changes:
```bash
supabase gen types typescript --linked > types/supabase.ts
```

## Request flow

```
Browser request
  → proxy.ts         (auth guard: unauthenticated → /login, authed → /dashboard)
  → App Router       (route groups: (auth), (student), (admin))
  → Server Component (default) or Client Component ("use client")
  → lib/supabase/server.ts (Server) or lib/supabase/client.ts (Client)
  → Supabase (DB + Auth + Storage)
```

shadcn/ui components are installed via `npx shadcn@latest add <name>` (style: base-nova, Tailwind v4 aware). Never edit `components/ui/` by hand.

## Tailwind v4 rules

- **No `tailwind.config.ts`** — configuration is CSS-only.
- Custom utilities come from `@theme inline` in `app/globals.css`.
- Available LMS utility classes: `bg-n-0` … `bg-n-950`, `text-n-0` … `text-n-950`, `bg-lms-accent`, `bg-lms-accent-50/100/600/700`, `text-lms-accent`, `bg-success`, `bg-warn`, `bg-danger`, `rounded-xs/sm/md/lg/xl`, `shadow-1`, `shadow-2`, `shadow-pop`.
- Dark mode: `darkMode` variant is `.dark` class on `<html>` (`@custom-variant dark (&:is(.dark *))`).

## Design system rules

See `lms-design-system.md` for full spec. Key rules:

- Background is `bg-n-0`, **not** `bg-white`.
- One accent color only: indigo `--a-500`. Used for CTA buttons, active states, focus rings.
- Headings: `font-semibold` (weight 600), `tracking-tight`.
- Body text: 14px base, `text-n-700`.
- All spacing multiples of 4px (`p-1` = 4px, `p-2` = 8px …).
- Borders: `1px solid` with `border-n-200` (hairline style).
- Border radii: buttons/inputs = `rounded-sm` (6px), cards = `rounded-md` (10px), modals = `rounded-lg` (14px).
- Shadows: use sparingly — `shadow-1` for cards, `shadow-2` for dropdowns, `shadow-pop` for modals.

## Important conventions

- `types/supabase.ts` is generated — never edit manually. Regenerate with `supabase gen types typescript --linked > types/supabase.ts`.
- Server Components are the default; add `"use client"` only where interactivity is needed.
- Route handlers live in `app/api/**/route.ts`.
- Supabase server client uses `cookies()` from `next/headers` — always `await` it in Next.js 16.

## Git conventions

- Use conventional commits: feat: fix: refactor: chore: docs:
- Branch naming: feat/* fix/* chore/*
- Never push to main directly, never push without explicit approval

## MVP scope

### Included
- Email/password auth — roles: student, admin
- Course catalog, module → lesson hierarchy
- Video lessons (Bunny.net) + watch_time progress
- Text lessons (MDX) + PDF materials (Supabase Storage)
- Single/multiple choice quizzes — server-side validation only
- Stripe one-time checkout + enrollment
- Student dashboard + admin content panel

### Excluded — do not suggest these
- Instructor role, certificates, comments, analytics
- Stripe subscriptions, discount codes, mobile app, AI features

## Security rules
- quiz_options.is_correct — NEVER send to client, check server-side only
- Bunny.net signed URLs expire in 24h — generate per-request
- RLS enforced: students see only their own enrollments and progress
- proxy.ts handles all route protection (Next.js 16 replaces middleware.ts with proxy.ts) — never trust client-side checks

### Known gotchas fixed
- `redirect()` inside Server Action + client `try/catch` — action returns `{ id }`, client does `router.push()`
- `Button asChild` not supported in this shadcn version — use `<Link className="...">` directly
- RLS on `profiles` needs `FOR SELECT TO authenticated USING (auth.uid() = id)` (not `FOR ALL`) for proxy to read role
- Event handlers in Server Component props — extract to a separate `"use client"` component (e.g. `DeleteCourseButton`)
- Base UI "uncontrolled FieldControl" warning — always use controlled state (`value` + `onChange`), never `defaultValue`
- `useRouter()` must be called inside the component body, not just imported
